## Engineering Values

**Simplicity.** Complexity emerges from poor design decisions. I prioritize clarity in code structure, naming conventions, and architectural patterns.

**Maintainability.** Code is read far more often than it is written. Systems should be easy to understand, modify, and extend by other engineers.

**Performance.** Efficient systems respect the user's time and device resources. Optimization is not optional.

**Accessibility.** Software built without considering diverse users and abilities is incomplete. Web applications must be usable by everyone.

**Testing.** Confidence in code comes from comprehensive, well-designed tests. I favor clarity and maintenance over coverage metrics.
